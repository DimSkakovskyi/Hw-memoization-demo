# React Memoization Demo

Цей проєкт демонструє, як **оптимізувати React-компоненти** за допомогою **мемоізації** — техніки, що дозволяє уникати зайвих обчислень і повторних ререндерів компонентів.

---

## Що реалізовано

У проєкті є два приклади:

### 1. `useMemo` — оптимізація обчислень

У компоненті **SquareCalculator** виконується обчислення квадрата числа.  
Завдяки **useMemo** результат зберігається і не перераховується при кожному ререндері, якщо саме число не змінилось.

> Без `useMemo`: кожна зміна теми чи іншого стану викликала б повторне обчислення.  
> З `useMemo`: React запам’ятовує результат, поки залежності не зміняться.

---

### 2. `useCallback` + `React.memo` — оптимізація функцій і дочірніх компонентів

У компоненті **Counter** кнопка `Increment` передається як дочірній елемент (`MemoButton`).  
Завдяки **useCallback**, функція `increment` не створюється наново при кожному ререндері.  
А **React.memo** не перемальовує кнопку, якщо її пропси не змінилися.

> Без `useCallback`: дочірній компонент ререндериться при будь-якій зміні стану.  
> З `useCallback` + `React.memo`: ререндер лише тоді, коли змінюється функція або пропси.
